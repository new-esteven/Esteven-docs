Cutscenes

STATUS: INCOMPLETE DOC, UNFINISHED SCENE
A scene used for loading Cutscenes, the cutsene being a json called "scenario" followed by a number.
All cutsenes/scenarios are stored in a folder (res://json n stuff/scenarios/)

json example

"backround": A path for the image that will be the background of the image.
"scenes": the number of dialogs (numbers) inside the json/cutscene/scenario.
"music": the background music that will be played when a json/cutscene/scenario loads.
"(any number)": data for each dialog, the number is the order.
"text": the text that will be displayed in the text box from the character. wheter if it is a thought or a character speaking.
"character": it can only be either "thought" or "character", otherwise the code will break. it refers wheter if youre gonna show the character node or the thought node (both a texturerect, with a label inside). image attached
"hidden": wheter to hide or show a character or thought in the screen. it can only be "true" or ""
"sprite": which sprite to show. put "false" if using thoughts. otherwise its should be a file path. list of character sprites
"audio": (optional) file path for a sfx if you want to use one at that dialog. it cannot be left empty otherwise the game will crash. if youre not gonna use any sfx, just dont put the "audio" tag.

node tree

dialog_scenario control node (the GUI one). contains everything.
character used to show any character sprite, hidden if youre using thoughts
label (any label) contains the text that the character/thought is saying.
thought used to only show the thought sprite. hidden if youre using characters
background the backgorund image for the cutscene.
bgm contains the background music that will be played when a json/cutscene/scenario loads. it changes its file automatically on the control_dialog.gd script.
sfx same shit that music but this time with sfx
